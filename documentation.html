Partie I — Optimiseur du code intermédiaire (opt.c)

Architecture générale

Le fichier opt.c implémente l'optimisation du code intermédiaire conforme au cours USTHB (SAYOUD Lynda). Il travaille directement sur le tableau global quad[] déclaré dans quad.h.

Principe fondamental
L'optimisation est appliquée en boucle jusqu'à point fixe : on répète les 6 passes tant qu'au moins une modification a eu lieu lors de l'itération précédente. On s'arrête au plus après 30 itérations.

Les 6 passes dans l'ordre d'exécution :

Passe 1 Simplification algébrique
Passe 2 Simplification des constantes chaînées
Passe 3 Propagation de copie
Passe 4 Propagation d'expression
Passe 5 Élimination des expressions redondantes (CSE)
Passe 6 Élimination de code inutile (Dead Code)

Fonctions utilitaires internes

Toutes les fonctions utilitaires sont déclarées static (visibles uniquement dans opt.c). Elles servent de briques de base réutilisées par toutes les passes.

est_nombre(s)

Rôle
Vérifie si la chaîne s représente un littéral numérique valide (entier ou flottant, avec signe optionnel). Exemple : "42", "-3.14", "+5" → 1 ; "x", "T0" → 0. 
/* Parcourt la chaine : signe optionnel, puis chiffres avec au plus un point */
static int est_nombre(const char *s)
{
    if (!s || !*s) return 0;
    const char *p = s;
    if (*p == '-' || *p == '+') p++;  /* signe optionnel */
    if (!*p) return 0;
    int a_point = 0;
    for (; *p; p++) {
        if (*p == '.') { if (a_point) return 0; a_point = 1; }
        else if (!isdigit((unsigned char)*p)) return 0;
    }
    return 1;
}

est_temporaire(s)

Vérifie si s est un temporaire généré par le compilateur : forme T0, T1, ... (T majuscule suivi d'au moins un chiffre). Cette fonction est cruciale : on n'élimine jamais une variable utilisateur même si elle semble inutilisée, car elle peut être observée de l'extérieur. On n'élimine que les temporaires.

est_saut(op)

Barrière de flot de contrôle
Retourne 1 si l'opérateur est BZ ou BR. Toutes les passes d'optimisation s'arrêtent devant un saut, car on ne peut pas garantir qu'une valeur reste valide si le flot peut bifurquer vers un autre chemin.

a_effet_de_bord(i)

Retourne 1 si le quadruplet numéro i produit un effet observable : sauts (BZ/BR), lecture clavier (input), affichage écran (out). Ces quadruplets ne sont jamais supprimés par l'élimination de code inutile, même si leur résultat semble mort.

compter_utilisations(var)

Parcourt tout le tableau quad[] et compte combien de fois var apparaît en op1 ou op2. Utilisé pour décider si un temporaire est encore consommé quelque part (propagation d'expression, élimination de code mort).

est_redefini_entre(var, debut, fin)

Vérifie si var est réaffectée (apparaît en res) dans l'intervalle de quadruplets [debut, fin[. Les sauts sont ignorés car ils ne définissent aucune variable. Utilisé pour vérifier la sûreté d'une propagation.

formater_nombre(v, buf)

Convertit un double en chaîne de caractères propre : si la valeur est entière (ex : 4.0), elle est affichée sans décimale ("4" et non "4.000000").

Passe 1 — Simplification algébrique

Référence cours : diapositive 105-106 (SAYOUD Lynda)

Principe
Remplacer des opérations coûteuses ou sans effet par des formes équivalentes plus simples, ou par leur résultat calculé directement à la compilation (constant folding).

Règles appliquées

AvantAprèsRègle
T = X + 0 T = X Élément neutre de l'addition
T = 0 + X T = X Commutativité de l'élément neutre
T = X - 0 T = X Élément neutre de la soustraction
T = X - X T = 0 Variable moins elle-même
T = X * 0 T = 0 Élément absorbant
T = X * 1 T = X Élément neutre de la multiplication
T = X * 2 T = X + X Addition moins coûteuse que multiplication (cours diapo 106)
T = X / 1 T = X Division neutre
T = C1 op C2 T = résultat Constant folding : deux constantes → calculer à la compilation

Techniquement, quand une règle s'applique, la passe transforme le quadruplet en une affectation simple (:=) : elle remplace l'opérateur par :=, copie la nouvelle valeur dans op1, et vide op2.

Avant optimisation
T0 = x * 1
T1 = y + 0
T2 = 3 + 5
Après simplification algébrique
T0 := x
T1 := y
T2 := 8

Passe 2 — Simplification des constantes chaînées

Référence cours : diapositive 109 (exemple t10 = j+1 ; t11 = t10-1)

Principe
Détecter deux quadruplets consécutifs qui font chacun une addition ou soustraction avec une constante, et dont le temporaire intermédiaire n'est utilisé qu'une seule fois. On fusionne les deux en un seul quadruplet avec la constante nette.

Pattern détecté

/* Pattern :
   quad[i] : T  = X  +/-  C1     (T utilisé exactement 1 fois)
   quad[j] : R  = T  +/-  C2
   ====>
   quad[j] : R  = X  +/- (C1 ± C2)   (combinaison nette)
   quad[i] : NOP                       (marqué à supprimer) */
Avant (exemple cours diapo 109)
t10 = j + 1
t11 = t10 - 1    ; C1=1, C2=1 → net=0
Après fusion (net = 1-1 = 0)
t11 := j          ; affectation directe
NOP               ; t10 marqué mort

Calcul du décalage net

op1op2netop_net
++C1+C2+
+-C1-C2signe(C1-C2)
-+C2-C1signe(C2-C1)
--C1+C2-

Si le net est exactement 0.0, le quadruplet d'usage devient une affectation directe R := X sans constante.

Condition de sécurité
La passe ne fusionne que si : le résultat du premier quad est un temporaire, utilisé exactement 1 fois, et qu'aucun saut ne se trouve entre les deux quads (un saut invalide la chaîne).

Passe 3 — Propagation de copie

Référence cours : diapositives 100-101

Principe
Quand on trouve dst := src, on remplace toutes les utilisations suivantes de dst par src, dans la mesure où ni dst ni src ne sont redéfinis entre-temps.
Avant
t1 := t2
t3 = 4 * t1
Après propagation
t1 := t2
t3 = 4 * t2   ← t1 remplacé par t2
Cas où la propagation est impossible (diapo 101)
t1 := t2
t1 := t3    ← t1 réécrasé : propagation t1=t2 invalide
t4 = t1 + t5
Ici, quand on arrive à la 3e ligne, la vraie valeur de t1 est t3 et non t2. La propagation s'arrête dès qu'on détecte que dst ou src est réaffecté.

Algorithme

for (i = 0; i < qc; i++) {
    if (quad[i].oper != ":=") continue;
    src = quad[i].op1;   /* valeur source */
    dst = quad[i].res;   /* variable destination */

    for (j = i+1; j < qc; j++) {
        if (est_saut(quad[j].oper)) break;       /* barrière */
        if (quad[j].res == dst) break;           /* dst réécrasé */
        if (quad[j].res == src) break;           /* src réécrasé */
        /* remplacer dst → src dans op1 et op2 */
        if (quad[j].op1 == dst) quad[j].op1 = src;
        if (quad[j].op2 == dst) quad[j].op2 = src;
    }
}

Passe 4 — Propagation d'expression

Référence cours : diapositive 102

Principe
Si un temporaire T est défini par T := src et n'est utilisé qu'une seule fois dans tout le code, on substitue directement src à l'endroit où T est utilisé, puis on supprime le quadruplet de définition (NOP).
Remarque importante du cours
Si T est utilisé plusieurs fois, on garde la variable pour ne pas recalculer l'expression à chaque usage.
Avant
T1 := expr     ; T1 utilisé 1 seule fois
T3 = 4 * T1
Après propagation d'expression
NOP
T3 = 4 * expr  ; T1 inliné directement

Conditions de sécurité

  • res doit être un temporaire (T0, T1, ...)
  • res doit être utilisé exactement 1 fois dans tout le code
  • Aucun saut entre la définition de T et son usage
  • src ne doit pas avoir été réaffecté entre la définition et l'usage

Passe 5 — Élimination des expressions redondantes (CSE)

Référence cours : diapositives 103-104

Principe — Common Subexpression Elimination
Maintenir une table des expressions disponibles. Quand la même expression (même opérateur, mêmes opérandes) est calculée une deuxième fois, remplacer le recalcul par une simple copie du résultat déjà stocké.
Avant (diapo 104)
t1 = a + b    ; mémorisé dans table
t2 = c * d
t3 = a + b    ; REDONDANT !
t4 = t3 * e
Après CSE
t1 = a + b
t2 = c * d
t3 := t1      ; copie, pas recalcul
t4 = t1 * e   ; après propagation de copie

Structure de la table des expressions

typedef struct {
    char operateur[100]; /* +, -, *, /, ==, ... */
    char operande1[100]; /* premier opérande   */
    char operande2[100]; /* second opérande    */
    char resultat [100]; /* temporaire résultat */
} ExpressionDisponible;

Gestion de la commutativité

Pour + et *, les opérandes sont interchangeables : a + b et b + a sont considérées identiques.

Invalidation de la table

  • Un saut (BZ/BR) vide entièrement la table : après un saut, on ne sait plus quelles valeurs sont encore valides.
  • Une réaffectation de res invalide toutes les entrées de la table qui dépendent de cette variable (en opérande ou en résultat).

Passe 6 — Élimination de code inutile (Dead Code)

Référence cours : diapositives 107-108

Principe
Supprimer physiquement du tableau quad[] les quadruplets dont le résultat n'aura aucun impact sur le programme final.

Deux catégories éliminées

  • Les NOP : marqueurs laissés par les passes précédentes (constantes chaînées, propagation d'expression).
  • Les temporaires morts : si res est un temporaire et que compter_utilisations(res) == 0, le calcul est inutile.
Ce qui n'est JAMAIS supprimé
Les quadruplets avec effet de bord : BZ, BR, input, out. On ne touche jamais aux sauts ni aux I/O.
Avant (diapo 107)
x   := t3       ; x jamais relu
a[t2] := t5
a[t4] := t3
Après élimination
a[t2] := t5     ; conservé
a[t4] := t3     ; conservé

Algorithme de compactage

/* 1. Marquer les quadruplets à supprimer */
for (i = 0; i < qc; i++) {
    if (a_effet_de_bord(i)) continue;
    if (strcmp(quad[i].oper, "NOP") == 0) { a_supprimer[i] = 1; continue; }
    if (est_temporaire(quad[i].res) && compter_utilisations(quad[i].res) == 0)
        a_supprimer[i] = 1;
}
/* 2. Compacter le tableau (décaler vers la gauche) */
int nouveau_qc = 0;
for (i = 0; i < qc; i++) {
    if (!a_supprimer[i]) quad[nouveau_qc++] = quad[i];
}
qc = nouveau_qc;
Important : après cette passe, les indices des quadruplets changent ! C'est pour cela que l'on doit renuméroter les cibles des sauts BZ/BR lors de la génération de code assembleur (voir système de labels).

Boucle jusqu'à point fixe

void optimiser_quadruplets(void)
{
    int modifie, iterations = 0;
    do {
        modifie  = simplification_algebrique();         /* Passe 1 */
        modifie |= simplification_constantes_chainees(); /* Passe 2 */
        modifie |= propagation_copie();                  /* Passe 3 */
        modifie |= propagation_expression();             /* Passe 4 */
        modifie |= elimination_expressions_redondantes(); /* Passe 5 */
        modifie |= elimination_code_inutile();           /* Passe 6 */
        iterations++;
    } while (modifie && iterations < 30);
    /* arrêt quand aucune passe n'a rien changé (point fixe) */
}
Pourquoi une boucle ?
Chaque passe peut débloquer de nouvelles opportunités pour les passes suivantes. Par exemple : la propagation de copie peut rendre visible une expression redondante qui n'était pas détectable avant. La boucle garantit qu'on atteint l'optimum local.

La limite de 30 itérations empêche une boucle infinie en cas de programme pathologique. En pratique, 2 à 4 itérations suffisent pour les programmes du cours.

Macro EMIT

#define EMIT(fmt, ...)  fprintf(f, fmt "\n", ##__VA_ARGS__)

Toute émission d'une ligne assembleur passe par cette macro pour garantir un retour à la ligne automatique et éviter la répétition.

Collecte des symboles (collecter_symboles)

Problème résolu
Le segment de données doit déclarer toutes les variables : celles de la table des symboles (TS) ET les temporaires générés par le compilateur qui ne sont pas dans la TS. Cette fonction construit une liste unifiée.

Structure SymInfo

typedef struct {
    char nom[128];    /* nom original (ex: "x", "T0", "Tabint") */
    char type[16];   /* "integer" ou "float"                   */
    char val[64];    /* valeur initiale concrète ou ""         */
    int  taille;     /* 0 = scalaire, >0 = tableau             */
    int  est_temp;   /* 1 = temporaire compilateur             */
} SymInfo;

Source 1 : Table des symboles (TS)

On parcourt les HASH_SIZE seaux de la table de hachage. Pour chaque nœud actif (state == 1) :

  • Si code == "TABLEAU" : la valeur dans val est la taille, pas une valeur initiale.
  • Si code == "CONST" : on récupère la valeur initiale stockée.
  • Si val == "oui" : c'est le marqueur interne de ts_marquer_init(), pas une vraie valeur → on passe à ? dans l'ASM.

Source 2 : Temporaires dans les quadruplets

On parcourt tous les quadruplets et leurs trois champs (op1, op2, res). Tout champ qui commence par T suivi de chiffres et qui n'est pas déjà dans la liste est ajouté comme temporaire (est_temp = 1).

Pour les accès tableaux de la forme "nom[indice]", la fonction est_tableau_access() extrait le nom et l'indice. Si l'indice est un temporaire, il est aussi déclaré.

nom_asm : renommage pour l'assembleur

/* T0 → _T0, T1 → _T1, ... (préfixe _ pour éviter conflits) */
static const char* nom_asm(const char *s)
{
    if (s[0] == 'T' && isdigit(s[1]))
        snprintf(buf, sizeof(buf), "_T%s", s + 1);
    return buf;
}

Les temporaires comme T6 deviennent _T6 dans l'ASM pour éviter tout conflit avec les mots réservés de l'assembleur.

Segment DONNEE (emettre_segment_donnees)

Référence cours : diapositives 137-141

Règles de déclaration

TypeDirective ASMExempleDescription
integer scalaire DW ? x DW ? 2 octets (word), non initialisé
integer initialisé DW val Max DW 100 2 octets, valeur concrète
float (simplifié) DW partie_entière Pi DW 3 Traité comme entier 16-bit (8086 sans co-processeur)
tableau d'entiers DW N DUP(?) Tabint DW 50 DUP(?) N mots de 2 octets, non initialisés
temporaire compilateur DW ? _T6 DW ? Non initialisé, recevra une valeur à l'exécution
tampon OUT DB 10 DUP(?), '$' _OUT_BUF DB ... Buffer d'affichage DOS, terminé par '$' (INT 21h AH=09h)
Cas particulier : val == "oui"
La table des symboles utilise ts_marquer_init() qui écrit la chaîne "oui" dans le champ val pour indiquer qu'une variable a été initialisée, mais sans stocker la valeur concrète. Dans le segment de données, ce cas est traité comme DW ? (non initialisé en ASM) car la valeur sera assignée par les instructions de code.

Utilitaires de chargement (charger/stocker)

Ces trois fonctions sont le cœur de la traduction. Elles encapsulent toute la logique de détection du type d'opérande (littéral, variable scalaire, accès tableau) et génèrent les instructions MOV appropriées.

charger_dans_ax(f, src)

Place la valeur de src dans le registre AX.

Type de srcInstructions générées
Nombre littéral (ex: "42") MOV AX, 42
Variable scalaire (ex: "x") MOV AX, x
Tableau + indice litéral (ex: "Tab[2]") MOV SI, 4   ; 2*2
MOV AX, Tab[SI]
Tableau + indice variable (ex: "Tab[i]") MOV SI, i
ADD SI, SI   ; SI = i*2
MOV AX, Tab[SI]
Pourquoi SI = i * 2 ? Chaque élément est déclaré DW (2 octets). L'adresse de l'élément i est donc à l'offset i × 2 depuis le début du tableau. On calcule SI = i*2 avec ADD SI, SI (= SI + SI = 2*SI), conformément à la diapositive 141.

charger_dans_bx(f, src)

Idem mais charge dans BX. Utilisé pour le second opérande des opérations binaires afin de ne pas écraser AX qui contient déjà le premier.

stocker_ax_dans(f, dst)

Écrit le contenu de AX dans la destination dst.

Procédures utilitaires dans LECODE

_PRINT_INT — Affichage d'un entier

Procédure NEAR appelée via CALL _PRINT_INT. Elle convertit la valeur dans AX en chaîne décimale ASCII, stockée dans _OUT_BUF, puis l'affiche via INT 21h AH=09h.

Algorithme de conversion entier → ASCII
  1. Sauvegarder tous les registres utilisés sur la pile (PUSH AX, BX, CX, DX, SI)
  2. Si AX < 0 : écrire '-', puis NEG AX
  3. Boucle _DIV_LOOP : diviser AX par 10, empiler le reste (chiffre ASCII)
  4. Boucle _POP_LOOP : dépiler les chiffres dans _OUT_BUF (ordre correct grâce au LIFO de la pile)
  5. Terminer par '$' (convention DOS)
  6. Appel INT 21h AH=09h pour afficher la chaîne
  7. Restaurer les registres (POP dans l'ordre inverse)
_PRINT_INT PROC NEAR
    PUSH AX | PUSH BX | PUSH CX | PUSH DX | PUSH SI
    CMP AX, 0
    JGE _PRINT_POS          ; AX positif → sauter
    MOV BYTE PTR [SI], '-'   ; sinon écrire le signe
    NEG AX
_DIV_LOOP:
    MOV DX, 0
    DIV BX                  ; AX=quotient, DX=reste (chiffre)
    ADD DL, '0'             ; convertir en ASCII
    PUSH DX                 ; empiler (LIFO → ordre inversé)
    INC CX
    CMP AX, 0
    JNE _DIV_LOOP
_POP_LOOP:
    POP DX                  ; dépiler dans l'ordre correct
    MOV [SI], DL
    INC SI
    LOOP _POP_LOOP
    MOV AH, 09h
    INT 21h                 ; afficher _OUT_BUF
    POP SI | POP DX | POP CX | POP BX | POP AX
    RET
_PRINT_INT ENDP

_READ_INT — Lecture d'un entier au clavier

Procédure NEAR appelée via CALL _READ_INT. Elle lit les caractères un à un via INT 21h AH=01h (lecture avec écho) jusqu'au CR (0Dh), et reconstruit l'entier en base 10.

Algorithme de lecture
  1. BX = accumulateur (résultat partiel), CX = flag signe
  2. Pour chaque caractère lu :
      – Si CR (0Dh) → fin
      – Si '-' → CX = 1 (négatif)
      – Sinon → chiffre : BX = BX * 10 + (AL - '0')
  3. Résultat dans AX. Si CX=1 → NEG AX.
_READ_CHAR:
    MOV AH, 01h
    INT 21h         ; AL = caractère lu
    CMP AL, 0Dh     ; Entrée ?
    JE  _READ_DONE
_READ_DIGIT:
    SUB AL, '0'     ; ASCII → chiffre
    CBW              ; étendre AL → AX
    XCHG AX, BX
    MUL 10           ; AX = BX * 10
    ADD AX, BX      ; AX = BX*10 + nouveau chiffre
    XCHG AX, BX
    JMP _READ_CHAR

Système de labels pour les sauts

Problème
Les quadruplets BZ et BR stockent leur cible sous forme d'indice dans le tableau quad[]. En ASM, on a besoin d'un label textuel à l'endroit correspondant.

label_pour(idx, buf)

Convertit un indice de quadruplet en label ASM : index 5 → "L5", index 12 → "L12".

marquer_cibles(marque[])

Construit un tableau booléen marque[qc] où marque[i] = 1 si l'indice i est une cible d'au moins un saut. On parcourt tous les quadruplets BZ et BR et on marque leur champ res (la cible). 

for (i = 0; i < qc; i++) {
    if (BZ or BR) {
        cible = atoi(quad[i].res);
        marque[cible] = 1;
    }
}

Utilisation lors de la traduction

Au début de la traduction de chaque quadruplet, si marque[idx] == 1, on émet le label L{idx}: avant les instructions. 

if (marque_label[idx]) {
    label_pour(idx, lbl);
    EMIT("%s:", lbl);   /* ex: L5: */
}

En fin de code, un label supplémentaire L{qc} est émis (après le dernier quad) pour que les BR qui sautent « à la fin du programme » aient une cible valide.

Traduction quadruplet par quadruplet

La fonction traduire_quadruplet(f, idx, marque_label) contient un if-else chain qui teste l'opérateur de chaque quadruplet et émet le code correspondant.

:= — Affectation

Diapositive 125

Quadruplet
(:=, a, , res)
ASM généré
MOV AX, a
MOV res, AX
Contrainte 8086 (diapo 132) : on ne peut pas faire MOV mémoire, mémoire directement. On passe toujours par un registre (AX).

+ — Addition (diapo 127)

Quadruplet
(+, a, b, res)
ASM généré
MOV AX, a
ADD AX, b    ; ou : MOV BX, b / ADD AX, BX
MOV res, AX

Si b est un nombre littéral, on utilise ADD AX, valeur (adressage immédiat). Sinon on charge d'abord dans BX.

- — Soustraction (diapo 127)

Quadruplet
(-, a, b, res)
ASM généré
MOV AX, a
SUB AX, b
MOV res, AX

* — Multiplication

Quadruplet
(*, a, b, res)
ASM généré
MOV AX, a
MOV BX, b
IMUL BX      ; DX:AX = AX * BX (signé)
MOV res, AX  ; on ne garde que les 16 bits bas
IMUL vs MUL
On utilise IMUL (multiplication signée) plutôt que MUL (multiplication non signée) pour gérer correctement les entiers négatifs du langage Prolang.

/ — Division

Quadruplet
(/, a, b, res)
ASM généré
MOV AX, a
MOV BX, b
CWD            ; étendre AX → DX:AX (signe)
IDIV BX        ; AX = quotient, DX = reste
MOV res, AX

CWD (Convert Word to Double Word) est indispensable avant IDIV pour préparer le dividende 32-bit dans DX:AX.

NEG — Négation unaire

MOV AX, a
NEG AX       ; AX = -AX (complément à 2)
MOV res, AX

==, !=, <, >, <=, >= — Comparaisons (diapo 128-130)

Les comparaisons produisent un résultat booléen (0 ou 1) stocké dans un temporaire. Le schéma général est : 

MOV AX, a
CMP AX, b         ; mettre à jour les flags
J{cond} _CMP{i}_V   ; si vrai, sauter à VRAI
MOV res, 0         ; sinon : résultat = 0 (faux)
JMP _CMP{i}_S      ; sauter la partie VRAI
_CMP{i}_V:
MOV res, 1         ; résultat = 1 (vrai)
_CMP{i}_S:         ; suite du programme
Opérateur ProlangCMP + saut
==JE
!=JNE
<JL (Jump if Lower, signé)
>JG (Jump if Greater, signé)
<=JLE
>=JGE

AND / OR — Logique booléenne

AND
MOV AX, a
MOV BX, b
AND AX, BX
MOV res, AX
OR
MOV AX, a
MOV BX, b
OR  AX, BX
MOV res, AX

NON — Négation logique

MOV AX, a
CMP AX, 0
JE  _NON{i}_U   ; si AX == 0, résultat = 1
MOV AX, 0
JMP _NON{i}_F
_NON{i}_U:
MOV AX, 1
_NON{i}_F:
MOV res, AX

BZ — Saut conditionnel (Jump if Zero) (diapo 130)

Quadruplet
(BZ, cond, , cible)
ASM généré
MOV AX, cond
CMP AX, 0
JE  L{cible}    ; si condition == 0 (fausse), sauter

Le compilateur Prolang stocke le résultat d'une condition (0=faux, 1=vrai) dans un temporaire. BZ saute si ce temporaire est 0 (condition fausse).

Quadruplet

Quadruplet

out — Affichage

  • Si l'opérande est une chaîne littérale (commence par ") : affichage caractère par caractère via INT 21h AH=02h.
  • Si l'opérande est une variable ou un nombre : charger dans AX puis CALL _PRINT_INT.

Structure complète du fichier .asm produit

Référence cours : diapositive 118

TITLE prolang.asm
; Code assembleur 8086 généré automatiquement

; ══ Segment de pile (diapo 133-136) ══════════════════════════

    DW 100 DUP (?)   ; 200 octets réservés
base_pile EQU $       ; étiquette → base de la pile
PILE ENDS

; ══ Segment de données (diapo 137-141) ═══════════════════════
DONNEE SEGMENT
    x                    DW ?
    Max                  DW 100
    Tabint               DW 50 DUP(?)
    _T0                  DW ?
    _OUT_BUF             DB 10 DUP(?), '$'
DONNEE ENDS

; ══ Segment de code (diapo 142-145) ══════════════════════════
LECODE SEGMENT

_PRINT_INT PROC NEAR  ; procédure affichage
    ...
_PRINT_INT ENDP

_READ_INT PROC NEAR   ; procédure lecture
    ...
_READ_INT ENDP

Debut:
    ASSUME CS:LECODE, DS:DONNEE, SS:PILE
    MOV AX, DONNEE
    MOV DS, AX        ; init DS
    MOV AX, PILE
    MOV SS, AX        ; init SS
    MOV SP, base_pile  ; SP = base (pile vide)

    ; ... instructions traduites depuis les quadruplets ...
L42:                   ; labels générés automatiquement
    MOV AH, 4Ch
    INT 21h           ; terminaison DOS (diapo 145)

LECODE ENDS
END Debut
Initialisation des segments (diapo 136)
On ne peut pas charger directement une valeur dans DS ou SS. On passe obligatoirement par AX :
MOV AX, DONNEE → MOV DS, AX et MOV AX, PILE → MOV SS, AX.

Partie III — Modifications apportées au projet existant

Fichier modifié : Syn.y

Deux modifications ont été apportées à Syn.y.

Modification 1 : Ajout de l'include asm8086.h

 #include "ts.h"

+#include "asm8086.h"    /* ← AJOUTÉ : génération code assembleur 8086 */
Pourquoi ?
Pour que la fonction generer_asm() soit accessible dans le fichier Syn.y (et donc dans le main() généré par Bison). Sans cet include, le compilateur C ne connaît pas la signature de la fonction.

Modification 2 : Appel de generer_asm() dans l'action du programme

Dans la règle Bison programme, après l'affichage du code intermédiaire optimisé, on appelle le générateur : 

             optimiser_quadruplets();
             printf("\n=== Code intermediaire APRES optimisation ===\n");
             afficher_qdr();
+            if (semantic_errors == 0) {
+                generer_asm("sortie.asm");  /* ← AJOUTÉ */
+            } else {
+                printf("Generation assembleur ignoree (erreurs semantiques).\n");
+            }
Condition semantic_errors == 0
On ne génère le fichier assembleur que si le programme source est sémantiquement correct. Si des erreurs sémantiques ont été détectées (variable non déclarée, type incompatible, etc.), la génération est ignorée car le code intermédiaire est probablement incohérent et produirait un ASM incorrect.
Nom du fichier de sortie : "sortie.asm"
Le fichier généré s'appelle sortie.asm et est créé dans le répertoire courant d'exécution. La fonction generer_asm() accepte un paramètre const char*, donc on pourrait facilement le rendre configurable.

Pourquoi ajouter asm8086.c ?
La chaîne de compilation GCC doit inclure tous les fichiers .c qui définissent des fonctions utilisées dans d'autres fichiers. asm8086.c définit generer_asm() appelée dans Syn.tab.c (le fichier C généré par Bison à partir de Syn.y). Sans cet ajout, l'éditeur de liens signalerait une erreur de symbole non défini (undefined reference to 'generer_asm').

Les 6 passes d'optimisation en une ligne

#PasseCe qu'elle faitExemple
1 Simplification algébrique Élimine les opérations sans effet, calcule les constantes X*1 → X, 3+5 → 8
2 Constantes chaînées Fusionne T=X+C1 suivi de R=T-C2 en R=X+(C1-C2) t=j+1; r=t-1 → r=j
3 Propagation de copie Remplace dst par src après dst:=src t1:=t2; 4*t1 → 4*t2
4 Propagation d'expression Inline T:=src là où T n'est utilisé qu'une fois T:=expr; 4*T → 4*expr
5 Expressions redondantes (CSE) Évite de recalculer une expression déjà calculée a+b calculé 2× → réutilise t1
6 Élimination code inutile Supprime les NOP et les temporaires jamais lus Compacte le tableau quad[]
