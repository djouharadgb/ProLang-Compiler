Les quadruplets sont des instructions de la forme (opérateur, op1, op2, résultat). L'optimiseur les transforme en place dans le tableau global quad[]. Le générateur ASM lit ensuite ce tableau optimisé pour produire le fichier .asm.
opt.c · Fonctions internes

Les fonctions utilitaires

Ces petites fonctions static sont utilisées par toutes les passes. Il faut les maîtriser car chaque passe en dépend.

est_nombre(s)
Vérifie si la chaîne représente un nombre (entier ou flottant, signe optionnel). Utilisé partout pour distinguer les littéraux des noms de variables.

Exemple : "42" → 1, "-3.5" → 1, "x" → 0, "T5" → 0
est_temporaire(s)
Vérifie si s a la forme T0, T1, T42... (T majuscule suivi de chiffres uniquement). Les temporaires sont créés par le compilateur, pas par le programmeur.

Exemple : "T12" → 1, "Tab" → 0, "i" → 0
est_saut(op)
Retourne 1 si l'opérateur est BZ (Branch if Zero) ou BR (Branch unconditional). Les sauts sont des barrières de contrôle de flot que l'on ne doit pas franchir lors de la propagation.
a_effet_de_bord(i)
Retourne 1 si le quadruplet i ne peut jamais être supprimé même si son résultat est inutilisé. Cas : sauts (BZ/BR), lecture clavier (input), affichage écran (out).
compter_utilisations(var)
Compte combien de fois la variable apparaît en op1 ou op2 dans tout le tableau de quadruplets. Utilisé pour savoir si un temporaire est mort (0 fois) ou à usage unique (1 fois).

Attention : ne voit PAS les indices de tableau dans les résultats (ex: Tab[T5]), d'où est_utilise_comme_index.
est_utilise_comme_index(var)
Cherche si var apparaît entre crochets dans un champ res, ex: Tab[T36]. Cela corrige le bug où compter_utilisations voyait 0 usages alors que T36 était indispensable.
est_redefini_entre(var, debut, fin)
Vérifie si var apparaît comme résultat (donc réécrite) dans les quads [debut, fin[. Utilisé avant une substitution pour s'assurer que la valeur n'a pas changé entre-temps.

Exemple : entre la définition de t1 et son usage, si x est réécrit, on ne peut pas substituer t1 par x.
formater_nombre(v, buf)
Convertit un double en chaîne. Si la valeur est entière (ex: 4.0), écrit "4" et non "4.000000". Utilise long long pour éviter les débordements sur les grandes valeurs.
nom_base_tableau(res, buf)
Si res vaut "Tabfloat[i]", extrait et retourne "Tabfloat". Nécessaire pour comparer correctement avec les variables lues dans op1/op2 (qui ne contiennent jamais les crochets).
opt.c

Phase d'optimisation — 8 passes itératives

La fonction optimiser_quadruplets() tourne en boucle jusqu'à ce qu'aucune passe ne modifie plus rien (ou 30 itérations max). Chaque passe retourne 1 si elle a changé quelque chose.

1
Propagation de constantes symboliques
propagation_constantes_symboliques()

Principe : Certaines variables sont déclarées CONST dans la table des symboles. Cette passe remplace leurs occurrences en op1/op2 par leur valeur numérique, afin que les passes suivantes puissent travailler sur des littéraux.


Comment ça marche :

  • Pour chaque quadruplet, examiner op1 et op2
  • Si le champ est un nom (pas un nombre) et que ts_est_constante(nom) est vraie
  • Récupérer la valeur via ts_get_val(nom)
  • Remplacer le nom par la valeur en mémoire (strcpy)
Exemple — avant/après
; Déclaration : const MAX := 100

; AVANT (quadruplets bruts)
(:=, 100, , MAX)
(<=, i, MAX, T5)

; APRÈS passe 1
(:=, 100, , MAX)
(<=, i, 100, T5)  ← MAX remplacé par 100
ℹ️
Cette passe ne modifie que les opérandes (op1, op2), jamais les résultats. L'affectation MAX := 100 sera supprimée plus tard par la passe 7 (variable morte).
2
Simplification algébrique
simplification_algebrique()

Principe : Appliquer des règles mathématiques pour simplifier les opérations sans changer le résultat. Si les deux opérandes sont des constantes, on calcule le résultat directement à la compilation (constant folding).


Règles appliquées :

OpérationRésultat
X + 0 ou 0 + XX
X - 0X
X - X0
X * 0 ou 0 * X0
X * 1 ou 1 * XX
X * 2 ou 2 * XX + X
X * 4X * 2 (→ relance)
X / 1X
X / X1
C1 op C2résultat calculé
Exemples de transformations
; X * 0 → 0
(*, n, 0, T3)
  ↓
(:=, 0, , T3)

; 3 * 4 → 12  (folding à la compilation)
(*, 3, 4, T5)
  ↓
(:=, 12, , T5)

; 2 * X → X + X  (addition = moins cher)
(*, 2, x, T7)
  ↓
(+, x, x, T7)

; X * 8 → X * 4 (puis relance → X * 2 → X+X)
(*, y, 8, T9)
  ↓
(*, y, 4, T9)
🐛
Bug corrigé — 2*X : L'ancienne version faisait strcpy(a,b); strcpy(b,a); ce qui écrasait a avant de le copier dans b. Résultat : b=b au lieu de b=X. Le fix utilise un tampon tmp.
3
Simplification des constantes chaînées
simplification_constantes_chainees()

Principe : Détecter un pattern où un temporaire T est défini par X +/- C1, puis immédiatement utilisé dans T +/- C2. Si T n'est utilisé qu'une fois, on fusionne les deux constantes en une seule opération.


Conditions requises :

  • Le 1er quad est + ou - avec une constante en op2
  • Le résultat res1 est un temporaire
  • compter_utilisations(res1) == 1
  • Pas de saut entre les deux quads
  • T n'est pas un indice de tableau FIX

Calcul du décalage net :

  • T = X+3, R = T+5 → R = X+8
  • T = X+7, R = T-2 → R = X+5
  • T = X-3, R = T+8 → R = X+5
  • T = X+1, R = T-1 → R = X (affectation directe)
Exemple complet
; AVANT
(+, j, 1, T10)   ← T10 = j + 1
(-, T10, 1, T11)  ← T11 = T10 - 1

; APRÈS  (1 - 1 = 0 → affectation directe)
(:=, j, , T11)    ← T11 = j
(NOP, , , )        ← T10 supprimé

; Autre exemple
(+, x, 3, T20)
(+, T20, 5, T21)
  ↓
(+, x, 8, T21)   ← 3+5=8
(NOP, , , )
4
Propagation de copie
propagation_copie()

Principe : Quand on voit dst := src, on remplace toutes les utilisations futures de dst par src directement. Cela élimine les affectations intermédiaires inutiles.


Arrêt de la propagation si :

  • Un saut est rencontré (barrière)
  • dst est réassigné
  • src est réassigné (la copie n'est plus valide)
⚠️
Bug corrigé — boucles : On ne propage QUE les temporaires en destination. Propager une variable utilisateur (i, x...) était dangereux : dans une boucle i := 1 puis i := i+1, remplacer tous les i par 1 casse la condition d'arrêt.
Exemple
; AVANT
(:=, T2, , T1)       ← T1 := T2
(*, 4, T1, T3)      ← T3 = 4 * T1
(+, T1, T3, T4)     ← T4 = T1 + T3

; APRÈS  (T1 remplacé par T2 partout)
(:=, T2, , T1)
(*, 4, T2, T3)      ← T2 à la place de T1
(+, T2, T3, T4)     ← T2 à la place de T1

; Puis passe 5 éliminera T1 := T2 si T1 = mort
5
Propagation d'expression
propagation_expression()

Principe : Si un temporaire T est défini par une affectation simple (T := src) et n'est utilisé qu'une seule fois, on substitue directement src à sa place d'utilisation, puis on supprime la définition de T (NOP).


Différence avec passe 4 :

  • Passe 4 : propage vers tous les usages suivants
  • Passe 5 : propage vers l'unique usage et supprime la définition

Conditions :

  • T est un temporaire
  • compter_utilisations(T) == 1
  • T n'est pas indice de tableau FIX
  • src n'est pas réassigné entre la définition et l'usage
Exemple
; AVANT
(:=, a, , T1)        ← T1 := a  (utilisé 1 seule fois)
(*, T1, b, T2)       ← T2 = T1 * b

; APRÈS
(NOP, , , )           ← définition supprimée
(*, a, b, T2)        ← T1 remplacé par a directement

; Cas indice tableau — PAS propagé
(:=, i, , T36)
(:=, v, , Tab[T36])  ← T36 est indice → garder
6
Élimination des expressions redondantes (CSE)
elimination_expressions_redondantes()

Principe : Common Subexpression Elimination. On maintient une table des expressions déjà calculées. Si la même expression réapparaît sans que ses opérandes aient changé, on remplace le recalcul par une copie du résultat précédent.


Structure ExpressionDisponible :

  • operateur, operande1, operande2
  • resultat : temporaire qui stocke la valeur

Règles d'invalidation :

  • Un saut vide toute la table
  • Une affectation res := ... invalide toutes les expressions contenant res
  • La commutativité est gérée pour + et * (a+b == b+a)
Exemple
; AVANT
(+, a, b, T1)   ← mémorisé : ("+",a,b) → T1
(*, c, d, T2)
(+, a, b, T3)   ← REDONDANT : a+b déjà dans T1
(*, T3, e, T4)

; APRÈS
(+, a, b, T1)
(*, c, d, T2)
(:=, T1, , T3)  ← T3 := T1  (copie, pas recalcul)
(*, T3, e, T4)

; Puis passe 5 ou 4 simplifiera T3 := T1

; Commutativité : b+a reconnu comme a+b
(+, b, a, T5)   ← T5 := T1 aussi
7
Élimination des variables mortes
elimination_variables_mortes()

Principe : Supprimer les affectations de variables utilisateur qui ne sont jamais lues dans tout le programme. Une variable est morte si elle n'apparaît jamais en op1 ou op2.


Approche conservatrice (corrigée) : On fait d'abord un scan global pour lister toutes les variables lues. Puis on supprime uniquement les affectations vers des variables absentes de cette liste.

🐛
Bug corrigé — boucles : L'ancienne version faisait une analyse en passe arrière linéaire. Dans une boucle, l'incrément i := i+1 était supprimé car au moment où il était scanné en arrière, i n'était pas encore marqué "vivant". La solution conservatrice résout ce problème.
🔧
Bug corrigé — tableaux : "Tabfloat[i]" en résultat ≠ "Tabfloat" en op1. On extrait le nom de base via nom_base_tableau() avant de tester.
Exemple
; Variable Pi déclarée mais jamais lue
(:=, 3, , Pi)
; scan global : Pi n'est dans aucun op1/op2
  ↓
(NOP, , , )   ← supprimé

; Variable i utilisée dans une boucle
(:=, 1, , i)          ← init
(<=, i, 10, T5)      ← i LU ici
(+, i, 1, T6)        ← i LU ici
(:=, T6, , i)         ← i dans la liste des lus → CONSERVÉ
8
Élimination de code inutile + Remappage des sauts
elimination_code_inutile()

Principe : Supprimer physiquement les NOP et les temporaires dont le résultat n'est jamais lu. Cette passe fait 3 étapes distinctes :


Étape 1 — Identifier les quads à supprimer :

  • NOP (marqués par les passes précédentes)
  • Temporaires avec 0 utilisations (jamais lus) et pas indice de tableau
  • Ne jamais supprimer les effets de bord (input, out, sauts)

Étape 2 — Construire la table de remappage FIX :
Avant de compacter, on calcule pour chaque quad ancien son nouvel index après compaction (remap[old] = new). On met ensuite à jour les cibles de tous les BZ/BR. Si la cible est elle-même supprimée, on avance au premier quad conservé.


Étape 2b — Supprimer les BR morts FIX :
Un BR dont la cible est l'instruction immédiatement suivante est un saut inutile. On le détecte après remappage et on le marque à supprimer.


Étape 3 — Compacter le tableau :
Copier les quads non supprimés vers les indices consécutifs. Mettre à jour qc (nombre de quads).

Exemple — remappage de saut
; AVANT compaction (indices originaux)
20: (<=, i, 10, T21)
21: (NOP, , , )              ← supprimé → remap[21] = -1
22: (BZ, T21, , 25)         ← cible 25 → recalcul nécessaire
23: (NOP, , , )              ← supprimé
24: (BR, , , 25)             ← après remap : cible = new(25) = 22+1 = mort
25: (+, i, 1, T26)

; Table remap : [20→20, 21→X, 22→21, 23→X, 24→22, 25→23]
; BZ 25 → BZ 23  ;  BR 25 → BR 23 (=22+1, mort)

; APRÈS compaction
20: (<=, i, 10, T21)
21: (BZ, T21, , 23)         ← cible mise à jour
22: [BR supprimé]
22: (+, i, 1, T26)          ← ex-indice 25
asm8086.c

Phase de génération assembleur 8086

La fonction generer_asm() traduit chaque quadruplet en instructions assembleur Intel 8086. Le code produit est un fichier .asm structuré en trois segments : pile, données, code.

Structure du fichier .asm généré
TITLE prolang.asm

PILE SEGMENT STACK           ; Segment de pile (100 mots)
    DW 100 DUP (?)
base_pile EQU $
PILE ENDS

DONNEE SEGMENT               ; Segment de données
    x                    DW ?
    _T5                  DW ?        ; temporaire
    Tabint               DW 10 DUP(?)  ; tableau
    _OUT_BUF             DB 12 DUP(?), '$'
DONNEE ENDS

LECODE SEGMENT               ; Segment de code
    ; ... procédures _PRINT_INT, _READ_INT ...
Debut:
    ASSUME CS:LECODE, DS:DONNEE, SS:PILE
    MOV AX, DONNEE
    MOV DS, AX          ; initialiser DS
    MOV AX, PILE
    MOV SS, AX          ; initialiser SS
    MOV SP, base_pile
    ; ... quadruplets traduits ...
Lqc:                          ; label de fin
    MOV AH, 4Ch
    INT 21h              ; terminer le programme DOS
LECODE ENDS
END Debut

Fonctions utilitaires (asm8086.c)

🔢 nom_asm(s)
Convertit un nom de variable en identifiant ASM valide. Les temporaires T5 deviennent _T5 pour éviter les conflits avec les registres ou directives 8086.
"T5"  →  "_T5"
"i"   →  "i"
"Tab" →  "Tab"
📦 est_tableau_access(s, ntab, idx)
Détecte la forme "Tab[indice]" et extrait le nom du tableau et l'indice dans des buffers séparés. Retourne 1 si c'est un accès tableau.
"Tabint[T5]"
  ntab = "Tabint"
  idx  = "T5"
  retour: 1
📥 charger_dans_reg(f, src, reg)
Charge la valeur de src dans AX ou BX. Trois cas : littéral numérique, accès tableau, variable/temporaire.
; src = "42"  (nombre)
MOV AX, 42

; src = "Tab[T5]"  (tableau)
MOV SI, _T5
ADD SI, SI       ; ×2 car DW = 2 octets
MOV AX, Tab[SI]

; src = "x"  (variable)
MOV AX, x
📤 stocker_ax_dans(f, dst)
Stocke le contenu de AX dans la destination dst. Deux cas : accès tableau ou variable simple.
; dst = "Tab[i]"
MOV SI, i
ADD SI, SI
MOV Tab[SI], AX

; dst = "x"
MOV x, AX

Procédures d'entrée/sortie

🖨️ _PRINT_INT (emettre_proc_print)
Affiche la valeur entière dans AX en décimal via DOS INT 21h. Algorithme :
1. Si AX < 0 : afficher '-' puis NEG AX
2. Diviser par 10 en boucle, empiler les restes (chiffres en ordre inverse)
3. Dépiler et afficher chaque chiffre avec INT 21h / AH=02h
4. Terminer le buffer avec '$' et appeler INT 21h / AH=09h
⌨️ _READ_INT (emettre_proc_input)
Lit un entier depuis le clavier caractère par caractère via INT 21h / AH=01h. Algorithme :
1. Accumulateur BX = 0, signe CX = 0
2. Si premier char est '-' : CX = 1
3. Pour chaque chiffre : BX = BX*10 + (char - '0')
4. À Entrée (0Dh) : AX = BX, si CX=1 → NEG AX

Traduction des quadruplets — traduire_quadruplet()

Affectation simple — (:=, a, , res)
; Quadruplet : (:=, x, , T5)
charger_dans_reg(f, "x", "AX");   →  MOV AX, x
stocker_ax_dans(f, "T5");        →  MOV _T5, AX
Addition / Soustraction — (+, a, b, res) ou (-, a, b, res)
; Quadruplet : (+, i, 1, T6)
MOV AX, i       ← charger a dans AX
ADD AX, 1       ← si b est un nombre : ADD AX, valeur directe
MOV _T6, AX    ← stocker le résultat

; Quadruplet : (+, x, y, T7)  (b = variable)
MOV AX, x
MOV BX, y       ← charger b dans BX
ADD AX, BX
MOV _T7, AX
Multiplication / Division — (*, a, b, res) ou (/, a, b, res)
; Quadruplet : (*, a, b, T8)
MOV AX, a
MOV BX, b       ← ou : MOV BX, valeur  si b est un nombre
IMUL BX         ← résultat dans DX:AX (on garde AX)
MOV _T8, AX

; Quadruplet : (/, a, b, T9)
MOV AX, a
CWD             ← étendre AX vers DX:AX (signe)
MOV BX, b
IDIV BX         ← quotient dans AX, reste dans DX
MOV _T9, AX

; CAS SPÉCIAL FIX — multiplication par flottant n.5 (ex: 1.5)
; x * 1.5  →  x * 3 / 2
MOV AX, x
MOV BX, 3       ← 2*1 + 1 = 3
IMUL BX
SAR AX, 1       ← division entière par 2 (shift arithmétique)
MOV _T8, AX
Comparaisons — (==, !=, <, >, <=, >=, a, b, res)
; Quadruplet : (<=, i, 10, T21)
; Stratégie : évaluer → 0 (faux) ou 1 (vrai)
MOV AX, i
CMP AX, 10         ← comparer
JLE _CMP7_V        ← si i<=10 : sauter au label "vrai"
MOV _T21, 0        ← résultat = faux
JMP _CMP7_S
_CMP7_V:
MOV _T21, 1        ← résultat = vrai
_CMP7_S:

; Table opérateur → mnémonique de saut
; ==  →  JE    |  !=  →  JNE
; <   →  JL    |  >   →  JG
; <=  →  JLE   |  >=  →  JGE
Sauts conditionnels et inconditionnels — BZ et BR
; Quadruplet : (BZ, T21, , 15)  — sauter si T21 == 0
MOV AX, _T21
CMP AX, 0
JE  L15            ← label généré : "L" + indice cible

; Quadruplet : (BR, , , 5)  — saut inconditionnel
JMP L5

; Les labels sont posés au bon endroit grâce à marquer_cibles()
; qui scanne tous les BZ/BR avant la traduction
L5:
    ; [5] (+, i, 1, T6)
    ...
Affichage (out) et lecture (input)
; Quadruplet : (out, "Bonjour", , )  — chaine littérale
; Chaque caractère affiché un par un via INT 21h / AH=02h
MOV AH, 02h  ;  'B'
MOV DL, 66
INT 21h
MOV AH, 02h  ;  'o'
MOV DL, 111
INT 21h
... (etc.)
; \n → emit_newline() → MOV DL, 0Ah + INT 21h

; Quadruplet : (out, x, , )  — variable
MOV AX, x
CALL _PRINT_INT    ← appel procédure d'affichage
MOV AH, 02h        ← saut de ligne après variable
MOV DL, 0Ah
INT 21h

; Quadruplet : (input, , , x)
CALL _READ_INT     ← résultat dans AX
MOV x, AX
Accès tableau — (TAB, Tabint, i, res)
; Quadruplet : (TAB, Tabint, i, T10)
; Chaque élément fait 2 octets (DW) → index × 2
MOV SI, i          ← charger l'indice
ADD SI, SI         ← SI = i × 2
MOV AX, Tabint[SI] ← lire l'élément
MOV _T10, AX       ← stocker dans le temporaire

; Quadruplet d'écriture : (:=, val, , Tabint[i])
; Géré par stocker_ax_dans avec est_tableau_access
MOV SI, i
ADD SI, SI
MOV Tabint[SI], AX
generer_asm()

Point d'entrée de la génération

La fonction generer_asm(nom_fichier) orchestre toute la génération en appelant les sous-fonctions dans le bon ordre.

Séquence d'appels dans generer_asm()
void generer_asm(const char *nom_fichier) {
    FILE *f = fopen(nom_fichier, "w");

    1. collecter_symboles();         ← scanner TS + quadruplets

    ; En-tête + segments pile et données
    2. EMIT("PILE SEGMENT STACK ...");
    3. emettre_segment_donnees(f);   ← toutes les variables en DW

    ; Segment de code
    4. emettre_proc_print(f);        ← procédure _PRINT_INT
    5. emettre_proc_input(f);        ← procédure _READ_INT

    ; Initialisation DOS (registres de segments)
    6. EMIT("Debut: ...");

    ; Marquage des cibles de saut
    7. marquer_cibles(marque);        ← tableau de flags

    ; Traduction de chaque quadruplet
    8. for (i = 0; i < qc; i++)
           traduire_quadruplet(f, i, marque);

    ; Fin du programme
    9. EMIT("L%d:", qc);              ← label de fin
       EMIT("MOV AH, 4Ch");
       EMIT("INT 21h");               ← terminaison DOS

    fclose(f);
}
🏷️
marquer_cibles(marque) : Avant de traduire, on scanne tous les BZ/BR pour identifier leurs cibles et poser un label (L5:, L12:...) au bon endroit. Le quad d'index 0 est toujours marqué (début du programme). Sans ce pré-scan, les labels seraient posés après le saut et non avant.
